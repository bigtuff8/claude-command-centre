Design decisions:
  - B001: Launcher health check + detached spawn. CRITICAL: add HTTP hooks to cloud template (claude-settings.json) to stop settings-sync from wiping them. 5s timeout on fire-and-forget hooks.
  - B002: Two-button modal — "Open Launcher" (primary, blue, opens terminal) + "Quick Session" (secondary, expands form for dashboard-managed). Replaces old single-form modal.
  - B003: pm2 for process management. npm scripts for start/stop/restart. Launcher detects pm2 availability, falls back to detached node spawn.
  - B004: Explicit 5s timeouts on all non-PreToolUse hooks. Enhanced reconnection banner with retry counter + manual button. Exponential backoff.
  - B005: Stop button (⏹) on active/waiting cards. Confirmation dialog. New "stopped" status (rose). Dashboard-managed = process kill, terminal = fallback message.
  - B006: Reframed as "hold" state (UI-level, not process-level). Pause icon in transcript header. Disables input bar, auto-declines permissions. Terminal sessions: greyed out.
  - B007: Token count + cost on card footer. Aggregate in metrics bar. Configurable pricing in config.json. "—" fallback if data unavailable. Developer must research data source first.
  - B008: New mobile breakpoint (<600px). Bottom tab bar. Full-width cards. Inline permission buttons. config.json host option (localhost/0.0.0.0/tailscale).
  - B009: Drop zone overlay on transcript panel. FileReader API reads content client-side. Prepended to next message as code block. 100KB limit, text files only, max 3 per message.

Prototype updated: prototype/index.html, styles.css, app.js
